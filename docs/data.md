# Data Preparation

## Overview

This document summarizes the data preparation requirements for NV-Generate-CTMR. For full details, see [../data/README.md](../data/README.md).

## Training Data Summary

NV-Generate-CTMR uses three types of training data:

### VAE Training Data

- **37,243 CT** training volumes and **1,963 CT** validation volumes from chest, abdomen, head and neck regions
- **17,887 MRI** training volumes and **940 MRI** validation volumes from brain, skull-stripped brain, chest, and below-abdomen regions
- Sources include TCIA, NLST, MSD, LIDC, and various OpenNeuro datasets

### Diffusion Model Training Data

- **10,277 CT** volumes from 24 distinct datasets
- Covers various body regions and disease patterns

### ControlNet Training Data

- **6,330 CT** volumes (5,058 training / 1,272 validation) across 20 datasets
- Covers different body regions and diseases

## Data Format

Training data should be in NIfTI format (`.nii.gz`). The ControlNet training workflow requires:

1. **Original images** (`*arterial*.nii.gz`)
2. **Encoded image embeddings** (`*arterial_emb*.nii.gz`) -- generated by the Autoencoder during preprocessing
3. **Original labels** (`mask*.nii.gz`)
4. **Pseudo labels** (`mask_pseudo_label*.nii.gz`) -- generated by [VISTA 3D](https://github.com/Project-MONAI/VISTA)
5. **Combined masks** (`mask_combined_label*.nii.gz`) -- combined original and pseudo labels

## JSON Data List Format

The training workflow requires a JSON file specifying image embedding and segmentation pairs:

```python
{
    "training": [
        {
            "image": "*/*arterial_emb*.nii.gz",
            "label": "*/mask_combined_label*.nii.gz",
            "dim": [512, 512, 512],
            "spacing": [1.0, 1.0, 1.0],
            "top_region_index": [0, 1, 0, 0],
            "bottom_region_index": [0, 0, 0, 1],
            "fold": 0
        },
        ...
    ]
}
```

## Example Preprocessed Dataset

An example preprocessed subset of [C4KC-KiTS](https://www.cancerimagingarchive.net/collection/c4kc-kits/) is available for download. See [../data/README.md](../data/README.md) for download links and details.

> **Note:** The example dataset references paths like `maisi/dataset/` for storing downloaded data. These paths are relative to the MONAI data directory (set via `MONAI_DATA_DIRECTORY` environment variable), not the repository root.
